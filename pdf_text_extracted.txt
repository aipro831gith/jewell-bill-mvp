Sri Narayan Jewellers — Template Spec
0. How To Use This Document
This document has two template specifications inside it: TEMPLATE A (G. Agarwal Chain) and TEMPLATE
B (Sri Narayan Jewellers). They use two different build methods because the source files are technically
different:
● TEMPLATE A source is a real vector PDF (has actual measurable text). Method: Rebuild using exact
coordinates below (Section 2).
● TEMPLATE B source is a flattened image (photo/scan, no measurable text layer). Method: Lock the
image as a background layer and print data on top at fixed coordinates (Section 3).
Do not attempt to freehand or "eyeball" recreate either design. Every number in this document was extracted by
measuring the original files directly. Use these numbers exactly.
Revision 2 — What Changed From The First Version
This version replaces the first draft. Three functional rules were added, one legal claim was corrected after
research (do not skip this section), and the HSN/turnover rule was confirmed accurate. Give this version to your
coder, not the earlier one.
● NEW — Section 1.5: Every buyer detail field, every item row, and every calculation figure is 100%
dynamic (changes per bill). Only the design/layout around them is fixed.
● NEW — Section 1.6: Purity column becomes fully optional — when switched off in the app, the
column disappears and neighboring columns (mainly Description) automatically stretch to fill the
space.

When a Discount column is added in a future version, the order of calculation must be locked as follows — this
is both a business-correctness rule and a tax-correctness rule:
Step Calculation
1 Item Amount (weight × rate) = Gross Item Value
2 Subtract Discount from Gross Item Value = Taxable Value
3 Apply CGST + SGST (or IGST) on the Taxable Value (after discount), NOT on the Gross Item
Value
4 Grand Total = Taxable Value + Tax
Applying GST before subtracting the discount would overcharge the customer on tax and would not match how
discounts should be treated for GST purposes — so this order must never be reversed, even accidentally, when
this feature is built. Flag this specifically to the coder when the discount feature development begins.
1.8 Legal findings — purity labeling for low-purity / alloy ornaments (researched)
This section documents research findings on a specific question: can jewellery below standard hallmark-eligible
purity be legally sold and invoiced, and how should it be labeled? Sources: current BIS and GST/CBIC-
notification-based reporting, checked as part of this project.
● BIS hallmarking (the official certified purity mark with a 6-digit HUID code) is currently only
available for these gold purities: 9K, 14K, 18K, 20K, 22K, 23K, and 24K. Gold below 9K purity has no
hallmark tier available at all — it legally cannot carry a BIS hallmark or HUID, regardless of how it's
labeled.
● Selling gold below 9K purity is not illegal. What matters is that the invoice must never claim a false or
invented karat number, and must never imply BIS certification that does not exist for that item.
● For silver: hallmarking remains voluntary nationwide in India. Any silver piece that IS hallmarked
must carry a HUID code. Silver that is not hallmarked can still be sold, but again must be labeled
honestly, with a real purity percentage, not a fabricated one.
● Confirmed approach for this app: for items that fall outside standard hallmark-eligible purity, the Purity
column switches from a Karat label (e.g. "22K") to a plain Percentage field (e.g. "37.5%") that the
business owner types in directly. The item description should use honest terminology — "Gold Alloy
Ornament" or "Silver Alloy Ornament" — rather than an invented karat figure. This keeps the invoice
legally accurate for both BIS and consumer-protection purposes.
● HSN code for these Alloy Ornament items: use the same base jewellery HSN chapter as regular
gold/silver ornaments — 7113 (or the fuller 6-digit code 711319 for unstudded articles of jewellery of
precious metal) — since alloy ornaments containing precious metal still fall under this same jewellery

4. PDF Output Quality & File Size Guidance
This applies to both templates. Goal: smallest possible file size with no visible quality loss — important for fast
loading, easy sharing (WhatsApp/email), and low storage cost at scale (thousands of invoices).
4.1 Rules to follow
● Generate all text, lines, boxes, and table borders as real vector objects (not as an image) — text and
shapes cost almost nothing in file size, unlike images.
● The ONLY image in the entire PDF should be the logo (Template A, if used as an image) and the
background form (Template B only). Never let the whole page render as one big image — that turns a
20KB file into a 1-2MB file for no visual benefit.
● Logo image: use PNG with transparency, kept under 50KB. Do not embed a huge camera-resolution
logo — resize it to roughly 300–500px on its longest side before embedding, since it only ever prints
small on the page.
● Template B background image: compress to roughly 150–200 DPI effective resolution for print (not
the full 1952×2752 camera resolution) — this is still sharp to the human eye on A4 paper but
meaningfully smaller in file size. Save as JPEG at ~80% quality, or PNG-8 if the design is mostly flat
colors.
● Never re-compress the same image multiple times through multiple tools — each pass loses quality.
Compress once, from the original high-resolution source, directly to the final target size.
● Fonts: embed only the specific font weights actually used (e.g. just Bold and Bold Italic, not the entire
font family) — this is called font subsetting and significantly reduces file size.
● Expected realistic output: Template A (fully vector) should be well under 100KB per PDF. Template B
(vector text + one background image) should realistically land in the 150–400KB range depending on
background compression — both are lightweight and fast to open/send.
